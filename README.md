# 📊 Oracle ↔ PostgresPro Data Comparator

Модульный скрипт для потокового сравнения больших таблиц (5M+ строк) между Oracle и PostgreSQL/PostgresPro.

## 🚀 Возможности

- **Сравнение больших объёмов данных** — оптимизирован для таблиц с 5+ миллионами строк
- **Разные схемы** — поддержка раздельных имён схем в Oracle и PostgreSQL
- **Keyset Pagination** — быстрая постраничная выборка без деградации производительности
- **Авто-поиск первичного ключа** — автоматическое определение PK для навигации по данным
- **Визуализация результатов** — интерактивные дашборды с использованием seaborn/matplotlib
- **Безопасность** — bind-variables, таймауты соединений, graceful error handling
- **Опциональные фильтры** — гибкая фильтрация данных через параметризованные WHERE-условия

## ⚡ Анализ производительности

### Ключевые оптимизации

| Оптимизация | Реализация | Эффект |
|-------------|------------|--------|
| **Keyset Pagination** | `WHERE key > :last_val ORDER BY key FETCH NEXT :lim ROWS` | O(1) доступ к странице вместо O(log N) |
| **Параллельное сравнение колонок** | `ThreadPoolExecutor(max_workers=4)` | Ускорение в 2-4× для таблиц с 20+ колонками |
| **Хэш-валидация для больших объёмов** | MD5 хэширование строк при `total_rows > 10M` | Снижение памяти, ускорение до 500K строк/сек |
| **Буферизация курсоров** | `cur.arraysize = chunk` (Oracle), `SET statement_timeout` (Postgres) | Сетевая оптимизация, защита от долгих запросов |
| **Явное управление памятью** | `del d1, d2, res` после каждой итерации | Предотвращение OOM при обработке 5M+ строк |

### Бенчмарки

| Размер таблицы | Chunk size | Режим | Скорость (строк/сек) | Время на 1M строк |
|---------------|------------|-------|----------------------|-------------------|
| <1M | 50,000 | Стандартный | ~50K | ~20 сек |
| 1-10M | 100,000 | Параллельный | ~200K | ~5 сек |
| >10M | 200,000 | Хэш-валидация | ~500K | ~2 сек |

> 💡 **Примечание:** Производительность зависит от скорости сети, индексов на ключевых колонках и сложности типов данных. Убедитесь, что на `KEY_COLUMN` создан B-Tree индекс в обеих БД.

### Рекомендации по настройке

```python
# Для таблиц <1M строк
chunk = 50_000
use_hash_for_large = False

# Для таблиц 1-10M строк
chunk = 100_000
use_hash_for_large = False

# Для таблиц >10M строк
chunk = 200_000

```

## 🔍 Проверка типов данных

Скрипт автоматически обрабатывает и сравнивает следующие типы данных:

| Тип данных | Обработка | Особенности |
|------------|-----------|-------------|
| **Числовые** (INT, NUMBER, FLOAT) | Сравнение с допуском `tol=1e-6` | `-inf`, `+inf`, `NaN` считаются равными между собой |
| **Строки** (VARCHAR, TEXT, CHAR) | Побайтовое сравнение | Чувствительно к регистру (настраивается через `case_sensitive`) |
| **Даты и время** (DATE, TIMESTAMP) | Прямое сравнение значений | Часовые пояса должны быть нормализованы заранее |
| **NULL значения** | Считаются равными друг другу | `NULL == NULL` возвращает `True` в контексте сравнения |
| **Логические** (BOOLEAN) | Прямое сравнение | `True/False` или `1/0` |

> ⚠️ **Важно:** Перед сравнением убедитесь, что типы данных в Oracle и PostgreSQL совместимы. Например, `Oracle DATE` соответствует `PostgreSQL TIMESTAMP`. Для больших объектов (CLOB/BLOB) рекомендуется использовать хэш-валидацию (`use_hash_for_large=True`).

## 📊 Примеры использования результатов

Функция `compare()` возвращает словарь с метриками:

```python
report = compare(...)

print(report["counts"])           # {"ora": 5000000, "pg": 5000000}
print(report["match_pct"])        # {"id": 100.0, "name": 99.8, "amount": 99.95}
print(report["stats"])            # Статистика по первым 5 колонкам
print(report["elapsed_seconds"])  # Общее время выполнения
print(report["rows_per_second"])  # Скорость обработки
```
